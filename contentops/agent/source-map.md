# Source Map (What Lives Where)

## GitHub Repo (Control Plane)

Store:

- prompts
- voice systems
- narrative assets
- templates
- schemas
- workflow specs

Do not store daily post operations here.

## Google Sheets (Operational DB)

Store:

- content ideas
- statuses
- reviewer notes
- publish dates
- URLs
- performance metrics

## Google Drive/Docs (Knowledge Sources)

Store:

- blogs
- transcripts
- founder notes
- decks
- research notes

## Slack (Review Layer)

Store:

- approval decisions
- revision rationale
- discussion context

## Website/Blog (External Source)

Store:

- published long-form material for repurposing

## Write Rules

1. Draft operations write to Sheet first.
2. Strategic assets write to repo only after review.
3. Reviewer decisions must be captured in Sheet.
